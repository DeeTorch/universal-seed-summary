What Is USS-ARCHIVE?

The Universal Seed Summary Invoker (USS) in ARCHIVE mode is a comprehensive protocol for preserving complete LLM conversation threads with full context, decisions, and resurrection capabilities across any AI model.

Core Innovation: USS-ARCHIVE transforms ephemeral AI conversations into permanent, portable, model-agnostic artifacts that capture not just what was said, but the complete context, decision trees, failure modes, and execution trajectory of an entire development session.

Architecture & Structure

USS-ARCHIVE employs a sophisticated 8-section schema designed to capture complete thread semantics. Tap any layer to expand details.

1. HEADER ‚Äî Thread Lock & Audit

Thread archetype, ignition vector, focus domains, depth metrics, drift events, completion state, momentum indicator, and finalization beacon. Establishes who initiated the thread, which model handled it, and what the thread accomplished.

2. FAILURE SEMANTICS & INTEGRITY FLAGS

Incoherence flags, compression loss warnings, inference boundary alerts, resolution impossibility markers, failure severity assessment, and compression fidelity score. Explicitly documents what was compressed, inferred, or left unresolved.

3. COSMIC CORE & EMERGENCE

Ontological constructs (protocols, schemas, engines), paradigm nodes (conceptual shifts between turns), and emergent universals ‚Äî novel capabilities or insights that arose during the session and were not pre-specified.

4. DECISIONS & GRAFTS

Architecture commits (what was decided), heuristic branches (what was proposed but not finalized), and epistemic locks (non-negotiable constraints). The complete decision ledger of the entire thread with full rationale.

5. OPEN VECTORS & THRUST

Unresolved queries, priority vectors (immediate / high / medium / low), and risk surfaces. Documents what remains incomplete and what the next session should tackle first.

6. THREAD TOPOLOGY

Parent threads, child threads, sibling threads, and cross-project links. Maps the thread's position in the larger Knowledge Graph using [[Wikilink]] format for Obsidian integration. NOTEBOOK keys render as #notebook/[key] tags.

7. EXECUTION ARTIFACTS

Generated outputs table (file, type, size, purpose), tool usage patterns, reusability index, and integration notes. Complete inventory of every file the thread produced, including size and deployment status.

8. INVOCATION LOCK & RESURRECTION HOOK ‚ú¶

Final immutability statement and the critical Resurrection Hook ‚Äî a > INGESTION: prefixed block (MUSS upgrades to > MUSS INGESTION:) containing compressed thread constraints, NOTEBOOK_STATE, and DECISIONS for seamless cross-model ingestion. Copy this block into any new LLM session to instantly resurrect full context.

How to Use USS-ARCHIVE

Integrate USS-ARCHIVE into your LLM workflow in four simple steps

30-Second Quick Start: Paste your USS-ARCHIVE .md artifact into any new LLM session. The model reads the > INGESTION: block at the bottom and instantly restores full thread context. That's it.
1

Invoke Archive Mode

At the end of any significant LLM development session, invoke USS in archive mode. Provide the thread context and request a comprehensive USS-ARCHIVE artifact. The model will generate a 3000‚Äì5000 token structured summary.

mode: archive protocol: "Universal Seed Summary Invoker" protocol_version: "1.3" thread_id: your-project-name-YYYYMMDD-archive project: "Your Project Name" invoker: "Your Name" target_model: "Model Name (e.g., Claude Sonnet 4)" timestamp: "YYYY-MM-DD HH:MM:SS TZ" token_target: 3000-5000 compression_ratio_estimate: "~14:1"
2

Save to Knowledge Base

Download the generated USS-ARCHIVE artifact as a .md file. Store it in your project's archives/ directory or Obsidian vault. The filename convention is: USS-[project]-[YYYYMMDD]-archive.md. Dropping it into Obsidian auto-populates your Knowledge Graph via [[Wikilinks]] in the THREAD TOPOLOGY section.

3

Commit to Repository

USS-ARCHIVE artifacts are immutable and repository-committable. Add them to your project's Git repository alongside code, documentation, and other artifacts.

# Commit message template git add archives/USS-[project]-[YYYYMMDD]-archive.md git commit -m "feat(archive): seal [project] thread ‚Äî USS v1.3" git push origin main
4

Resurrect When Needed

To continue a thread in any LLM, copy the entire USS-ARCHIVE artifact into a new session. The model will parse the Resurrection Hook (the > INGESTION: block at the end) and immediately restore full thread context, constraints, and decision state.

Cross-Model Compatibility: The same USS-ARCHIVE artifact works in Claude, GPT-4o, Gemini, Grok, and Perplexity without modification. The Resurrection Hook provides universal model ingestion across all five wrappers.

MUSS v1.0: The Full Lifecycle Evolution

USS-ARCHIVE has evolved into MUSS (Memory-Augmented Universal Seed System) for complete session lifecycle management

USS v1.3: End-of-Session Archival

Original protocol. Invoked at thread completion. Generates immutable archive artifact with 8-section schema. Resurrection Hook prefix: > INGESTION:. No live session governance.

MARM v1.4: Live Session Governance

Complementary protocol for active sessions. Anti-drift commands (/m remember, /m focus). Session-scoped only. No archival schema. Governs the live session ‚Äî the half USS leaves unaddressed.

MUSS v1.0: Unified Lifecycle Engine ‚ú¶

Blended USS √ó MARM. Governs ACTIVE ‚Üí SEED ‚Üí ARCHIVE ‚Üí RESURRECT phases. Embeds LIVE MEMORY SNAPSHOT (NOTEBOOK_STATE + SESSION_LOG_DIGEST) in every artifact ‚Äî absent from USS v1.3. 9-section schema. 5 model-specific wrappers: Claude (XML), GPT-4o (Markdown hierarchy), Grok (terminal/directive), Gemini (procedural + STEP 0 silent guard), Perplexity (direct Markdown). Bilaterally validated: test_muss_protocol.py (72 assertions, T1‚ÄìT10) + muss_validator.py (artifact compliance layer). Resurrection Hook upgrades to > MUSS INGESTION: superset.
