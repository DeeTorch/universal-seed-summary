# USS Engine v1.2 Roadmap

Theme: USS Engine v1.2 — Semantic Evidence + Archive Store

This document defines the next release direction only. Do not start v1.2 implementation until v1.1.1 is tagged, release notes are published, and provider validation/adoption docs are complete.

## Proposed Scope

- Semantic evidence matching.
- Archive store for USS artifacts.
- Artifact index/search.
- Provider validation matrix automation.
- GitHub issue/PR thread summarizer.
- Obsidian export integration.
- Pre-commit secret scanning.
- Optional package distribution workflow.

## Guardrails

- Do not add new provider clients as part of roadmap drafting.
- Do not rewrite the core engine for speculative architecture.
- Prefer issues with clear acceptance criteria over broad architecture documents.
- Keep security and release validation work ahead of feature expansion.

## Issue Drafts

- Add semantic evidence matching with measurable precision/recall examples.
- Design an archive store for generated USS artifacts with stable metadata.
- Add artifact index/search for local archive queries.
- Automate provider validation matrix updates from smoke and checkpoint results.
- Summarize GitHub issue and PR threads into USS-compatible inputs.
- Export validated summaries into Obsidian-friendly Markdown.
- Add pre-commit secret scanning and generated-output deny rules.
- Evaluate package distribution workflow for tagged releases.
